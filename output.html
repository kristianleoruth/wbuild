Getting started

This is a simple custom markup language (inspired by Markdown). The script wbuild.py is a converter allowing one to quickly build a site from a markup file. The structure of the markup document is a series of tags with text in between. These tags represent a section, image, text, header, subheader, etc, and are supplemented with arguments given in extra square brackets following the tag.

To convert a text document to an html file, run the command:

# Pass arguments:
# mode: light or dark, optional
# view: open website after generation, optional
python3 wbuild.py "path_to_doc.txt" "output_path.html" -mode -view
# Demo (build this website):
python3 wbuild.py

For examples of the markup language, see the Examples section.

Created 5 March 2025 by Kristan Leo Ruth <krirut@kth.se, kristianleoruth@gmail.com>.

Tags

Tags are formatted as [tag_name] and represent components of the site, e.g. a section, column, image, text, etc. The square brackets may be used for plain text when inside {} and part of a command argument, or when the brackets are escaped with \.

Definitions below of tags give the arguments and their default values for the given tag (omitting arguments from a tag results in default values being used). For example, [section][align=justify, notopmarg=False] is equivalent to [section] because the arguments given are the same as default.

Any tag may be passed a string argument uid (unique), which may be referenced in e.g. links. Additionally, a class argument may be passed to any tag.

Sections

The document (doc) is a special [section] containing a collection of [section] tags. After establishing a section with a tag, any following tags will belong to the section, until a new secton is started. Other than the doc, sections may not hold [section] tags. Adding a new [section] tag will end the previous section.

Definition:

{
    "type": "section",
    "args": {
        "align":"justify",
        "bg": 1,
        "class": "",
        "notopmarg": False,
        "uid": "",
    }
}

Example use:

[section]
[header][label=This is a header]
This text will be part of the section.

[section][align=center]
[header][label=New centered section]
This is a new section, the content within is aligned to the center.

Columns

The tag [column] allows the user to define a new column. Adding column tags will automatically convert the parent section into a grid layout based on the number of columns (maximum 3). The column may also hold a list of items, so new tags will be children of the column.

Creating a new column will end the previous one, as with sections. Remember that sections cannot be children of sections, only of the doc.

Definition:

{
    "type": "column",
    "args": {
        "align":"justify",
        "bg": 1,
        "class": "",
        "notopmarg": True,
        "uid": "",
    }
}

Example use:

[section]
[column]
This text will appear on the left column
[column]
This text will appear on the right column
[section]
This will start a new section

Header/subheader

Use [header], [subheader], or [subsubheader] with a second tag identifying the label argument to create headers and subheaders of various sizes.

Definition (shared among sub/headers except for "type"):

{
    "type": "header",
    "args": {
        "label": "",
        "class": "",
        "notopmarg": False,
        "uid": "",
    }
}

Example use:

[header][label=Some header text]
[subheader][label=Subheader text]
[subsubheader][label=Even smaller subheader]

Text

Text that is not in unescaped square brackets, i.e. [] is automatically added to the preceding section/column. To pass arguments to the text tag, declare a [text] tag and then pass the arguments.

Definition:

{
    "type": "text",
    "args": {
        "align": "justify",
        "bg": 1,
        "class": "",
        "notopmarg": False,
        "uid": "",
    },
}

Example use:

[section]
This text will automatically be in the above declared section.
\italic{This text will be italic.}
\link{https://www.google.com/}{This text will be a link to Google.}
\bold{This text will be bold} and this text won't be.
\textcode{This text will look like code}

[text][class=some_class some_other_class]
This text will be separate from the earlier text and will have classes 'some_class' and 'some_other_class'

Displaying code

Code can be displayed inline or in a block on a new line. Use the textcode command or the [code] tag. For code blocks use the format [code][args: optional][your_code].

Definition:

{
    "type": "code",
    "args": {
        "align": "left",
        "class": "",
        "notopmarg": False,
        "uid": "",
    }
}

Example use:

[section]
[code][
This code will be displayed in a block.

More lines can be displayed this way.
]

Lists

Lists can have ordered and unordered items. Ordered items are numbered (e.g. 1.2.1) and unordered items are bulleted or have no symbol. Use argument orderall=True to count unordered items in the ordered item numbering. baseoffset determines the offset (number of spaces before symbol) for first level items and lvloffset is the added offset per level of depth in the list. Lists have a maximum depth of 3, i.e. up to three indicators (*/-/#) before the closing parenthesis. An example of a list is the table of contents.

Definition:

{
    "type": "list",
    "args": {
        "orderall": False,
        "lvloffset": 4,
        "baseoffset": 2,
        "class": "",
        "notopmarg": False,
        "uid": "",
    }
}

Example use:

[list][
Unordered items, bulleted symbol:
*) First level unordered items
**) Second level unordered items
***) Third level unordered items

Unordered items, no symbol:
-) First level unordered items
--) Second level unordered items
---) Third level unordered items

Ordered items, int(.int.int) symbol:
#) First level ordered items
##) Second level ordered items
###) Third level ordered items
]

Images

Images and gifs may be added by the [img] tag and the path is provided as the src argument. These can be attached to sections and columns. Caption may be added via the caption argument. Captions may take the same commands as text (i.e. links, bold, italic, textcode, etc.).

Definition:

{
    "type": "img",
    "args": {
        "src": "", # "/dir_path/image.png"
        "maxwidth": "100%", # css value
        "caption": "",
        "label": "",
        "italicize":True,
        "class":"",
        "notopmarg":False,
        "uid":"",
    }
}

Example use:

[section][align=center]
[img][
    src=./path_to_img.gif, 
    maxwidth=50%, 
    caption=This caption and image will be centered due to the section argument.
]

In-text commands

Commands are given inside text meant to be displayed in tags (excluding [code]). This includes [text], [subsub/sub/header], [img], [list], etc. Commands are given in the form \keyword{arg1}{arg2} with one or two arguments depending on the command.

Modifying text appearance

Boldness and italics can be applied to text via \bold{} and \italic{}:

\italic{This text will be italic,} and \bold{this text will be bold.}

Display arguments

To display an argument within text, use the \showargs{uid}{arg} command. This will replace the command text with the (to string) value of the argument.

[section][uid=ex1, notopmarg=True]
\showarg{ex1}{notopmarg} -> "True"

Referencing labels

It's possible to reference some image as, for example, figure 1 if the image's label argument is set appropriately. This will automatically apply a link to that image with the text being the value of the label argument. This goes for any tag that accepts a label argument. It is recommended to keep labels to numbers if they are for figures (as show below), except for headers and subheaders as the label determines the text displayed. Labels do not need to be unique.

[img][src=some_path.png, uid=some_uid, label=1]
As we can see in figure \showarg{some_uid}{label}, ...
\showarg{some_uid}{label} -> "1"

[header][uid=some_header, label=Linked header]
\showarg{some_header}{label} -> "Linked header" # this is a link to the header

To have an image display the caption with preceding "Figure X: ", one could set the label to some number X, then set the caption as:

[img][
uid=example_fig, 
label=1, 
caption=Figure \showarg{img_uid}{label}: your caption.
] -> caption=Figure 1: your caption.

Linking to sections

Link to parent section In-text commands. In-page references to other tags are done by passing a string uid argument to the tag, then passing #uid as the link.

[section][uid=link_to]
\link{#link_to}{This is a link to this section.}

Inline code

To display code inline, use the \textcode{your code} command.

[section]
This text is part of the section and \textcode{this code is inlined.}

List of all commands and arguments

Arguments can be optional if listed as {arg: opt.}.

\bold{text to bold}
\italic{text to italicize}
\link{url}{text: opt.}
\textcode{inline code}
\showarg{uid}{argname}
